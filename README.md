# Team Project Version Control

## Repository Setup

A GitHub repository was created to demonstrate version control using Git and GitHub.

## Team Modules

* Login Module
* Payment Module
* Report Module

## Branch Strategy

* main
* login-module
* payment-module
* report-module

## Workflow Followed

1. Created a GitHub repository.
2. Cloned the repository to the local machine.
3. Created separate branches for each module.
4. Added module files and committed changes.
5. Pushed branches to GitHub.
6. Created Pull Requests for each branch.
7. Reviewed and merged Pull Requests using merge commits.
8. Managed version control using Git branching and merging.

## Pull Requests

* PR #1 – Added login module
* PR #2 – Added payment module
* PR #3 – Added report module

## Conflict Resolution

Git merge conflicts can occur when multiple branches modify the same section of a file. Conflicts are resolved by manually editing the file, keeping the required content, and committing the resolved version.

## Conclusion

This project successfully demonstrated repository creation, branching, committing, pushing, pull requests, merging, and collaborative version control using Git and GitHub.
